## Conventions

<!-- Project house rules + AI-maintenance preferences, in plain prose.

每次完成新功能或 Bug 修复并验证后创建 Git 提交。提交信息采用 Gitmoji 格式：以 https://gitmoji.js.org/ 对应的 emoji 开头，后接简洁的中文改动说明。默认只提交，不自动推送远端。

编写代码时不要创建常量，除非同一个值或表达式被复用 3 次或以上。

不要将代码全部写在一个文件中；按功能或职责拆分文件，并保持模块边界清晰。

代码编写必须简洁，并以新手容易理解和维护为优先。
